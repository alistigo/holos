# Task 006 — Persistence: LocalStorageListRepository

## Goal

Implement the `LocalStorageListRepository` in `packages/alistigo-widget` (infrastructure layer), wire it into `ListApplicationService`, and verify that List state survives a page reload.

## Reference

- PRD F5 (persist across reloads), N2 (no backend)
- [Domain model — Repository Interface section](../../../projects/alistigo-ai/domain/index.md)

## Deliverables

```
packages/alistigo-widget/src/
  infrastructure/
    local-storage-list-repository.ts   # implements ListRepository
```

### LocalStorageListRepository

```typescript
class LocalStorageListRepository implements ListRepository {
  private readonly keyPrefix = 'alistigo:list:'

  async load(id: ListId): Promise<List | undefined>
  async save(list: List): Promise<void>
}
```

- Storage key: `alistigo:list:<ListId>`
- Storage value: `JSON.stringify(AlistigoDocument)`
- On load: `DocumentSerializer.deserialize(JSON.parse(raw))`
- On save: `localStorage.setItem(key, JSON.stringify(DocumentSerializer.serialize(list)))`
- If `localStorage` is unavailable (SSR, private browsing restriction): log warning, degrade gracefully — `load` returns `undefined`, `save` is a no-op

### Wiring

- `<alistigo-list>` instantiates `ListApplicationService` with `LocalStorageListRepository`
- On mount: if `list-id` attribute present, call `loadDocument(listId)` and render
- On mutation: `save` is called automatically after each command inside `ListApplicationService`

## Acceptance

- Add elements, reload page → same elements present
- Delete element, reload page → deleted element absent
- Storage key format is `alistigo:list:<uuid>`
- localStorage unavailability does not throw — widget renders with empty/initial state
