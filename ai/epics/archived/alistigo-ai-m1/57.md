# Task 001 — Domain core: TypeScript DDD model

## Goal

Scaffold `packages/alistigo-domain` and implement all DDD building blocks defined in `projects/alistigo-ai/domain/index.md`. This package is a zero-infrastructure, pure-TypeScript domain layer. No UI, no I/O, no localStorage.

## Reference

[Domain model](../../../projects/alistigo-ai/domain/index.md)

## Dependencies

- `typeid-js` — TypeID generation and validation (`typeid("lst")`, `TypeID.fromString()`)

## Deliverables

### Package scaffold
```
packages/alistigo-domain/
  src/
    value-objects/
      list-id.ts              # ListId: TypeID<"lst">
      list-element-id.ts      # ListElementId: TypeID<"lse">
      list-event-id.ts        # ListEventId: TypeID<"lev">
      list-element-content.ts # ListElementContent (non-empty, trimmed, max 2000 chars)
      actor.ts                # Actor: 'user' | 'llm' | 'host' | 'system'
      timestamp.ts            # Timestamp (ISO 8601 UTC string VO)
      schema-version.ts       # SchemaVersion (semver string)
    events/
      list-created.ts
      list-element-added.ts

```

### Key behaviors to implement

- `List.create(cmd: CreateList)` — factory, emits `ListCreated`; generates `listId` if not provided
- `List.addListElement(cmd: AddListElement)` — validates non-empty content, assigns `ListElementId` via `typeid("lse")`, emits `ListElementAdded`
- `List.deleteListElement(cmd: DeleteListElement)` — validates element exists, emits `ListElementDeleted`
- `List.exportListDocument(cmd: ExportListDocument)` — emits `ListExported`, returns event (serialization handled by document-format package)
- `List.applyEvent(event: ListEvent)` — private; mutates state from a single event (used for rehydration)
- `List.getUncommittedEvents()` — returns events since last load

### Invariants to enforce (throw `ListError` if violated)

- `AddListElement`: `ListElementContent` must be non-empty after trimming
- `DeleteListElement`: `listElementId` must exist in current elements

## Acceptance

- Unit tests cover all invariants
- `List.create → addListElement → deleteListElement` round-trip produces correct element state
- IDs are TypeID instances: `ListId` prefix `lst`, `ListElementId` prefix `lse`, `ListEventId` prefix `lev`
- No imports from `infrastructure`, `application`, or UI layers
