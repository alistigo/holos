# Task 005 — Widget: embeddable web component

## Goal

Build `packages/alistigo-widget` — the self-contained, embeddable `<alistigo-list>` web component that renders a List, handles add/delete interactions, and dispatches commands through `ListApplicationService`.

## Reference

- PRD F2 (render), F3 (add), F4 (delete), N1 (embeddable), N2 (no backend)
- [Domain model — Application Services](../../../projects/alistigo-ai/domain/index.md)

## Deliverables

```
packages/alistigo-widget/
  src/
    components/
      alistigo-list.ts       # <alistigo-list> custom element
      element-list.ts        # renders Element[]
      element-row.ts         # single element + delete button
      add-element-form.ts    # text input + add button
    widget-controller.ts     # bridges DOM events to ListApplicationService
    index.ts                 # registers custom elements
  public/
    alistigo-widget.js       # bundled IIFE output (embeddable via <script>)
```

### `<alistigo-list>` attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `list-id` | string | `ListId` to load from localStorage |
| `document` | string | JSON-serialized `AlistigoDocument` (overrides localStorage lookup) |
| `actor` | string | Default actor for commands (`user` if omitted) |

### Rendering contract

- Empty list: renders container with no element rows and a visible add form
- Non-empty list: renders elements in insertion order, each with a delete button
- After add: new element appears at the bottom
- After delete: element disappears; remaining order unchanged

### Event handling

- Add form submit → `AddElement` command → re-render
- Delete button click → `DeleteElement` command → re-render
- All DOM updates synchronous within the same microtask where possible

### Embeddability

- Self-contained: no external CSS frameworks, no global state leakage
- Works inside a shadow DOM boundary
- Registers as `<alistigo-list>` via `customElements.define`

## Acceptance

- Renders correctly in a real browser (Playwright screenshot validation)
- Add + delete interactions update the UI without page reload
- No console errors on mount with a valid `list-id` or `document` attribute
