# Task: Scaffold @alistigo/artifact-config-list-format

## Description

Create `packages/alistigo-artifact-config-list-format/` — the leaf package that defines
the JSON Schema and TypeScript types for the list artifact's config document.

This is a **leaf** package with no `@alistigo` dependencies.

## Acceptance Criteria

- [ ] Package scaffolded via `bash scripts/new-package.sh alistigo-artifact-config-list-format`
- [ ] `package.json` name: `@alistigo/artifact-config-list-format`; no `@alistigo` dependencies
- [ ] JSON Schema defined for list config:
  ```json
  {
    "$schema": "http://json-schema.org/draft-07/schema#",
    "type": "object",
    "properties": {
      "readonly": { "type": "boolean", "default": false }
    },
    "additionalProperties": false
  }
  ```
- [ ] TypeScript types exported: `ListArtifactConfig { readonly?: boolean }`
- [ ] Validation helper exported: `validateListConfig(value: unknown): ListArtifactConfig`
- [ ] Unit tests covering valid config, missing `readonly` (defaults to false), extra fields rejected
- [ ] `pnpm build` and `pnpm test` pass for this package

## Technical Details

The config schema is a partial — it describes only the list-specific fields. The base
fields (`app`, `lang`) are owned by `@alistigo/artifact-config-format`.

Export from `src/index.ts`:
- `listArtifactConfigSchema` — raw JSON Schema object
- `ListArtifactConfig` — TypeScript type
- `validateListConfig` — validates and returns typed config (throws on invalid)
