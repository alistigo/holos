# Task: Scaffold @alistigo/artifact-manager

## Description

Create `packages/alistigo-artifact-manager/` — the package that reads the config
document from the host page, validates it, resolves the artifact's CDN URL, and injects
the script tag that boots the artifact.

Exported as both UMD (self-contained, for use in the browser without a bundler) and ESM
(for tooling/playground).

## Acceptance Criteria

- [ ] Package scaffolded via `bash scripts/new-package.sh alistigo-artifact-manager`
- [ ] `package.json` name: `@alistigo/artifact-manager`; depends on `@alistigo/artifact-config-format`
- [ ] Internal artifact map (seed entry):
  ```ts
  const ARTIFACT_MAP: Record<string, string> = {
    "@alistigo/artifact-list": "<cdn-umd-url-from-m1-deployment>"
  }
  ```
- [ ] `initArtifactManager(config: ArtifactConfig): void` — validates config, looks up CDN URL, injects `<script>` tag into `document.head`
- [ ] Throws clear error if `config.app` not found in artifact map
- [ ] UMD build: single self-contained file, no external runtime dependencies
- [ ] ESM build: standard `dist/index.js` + `dist/index.d.ts`
- [ ] Unit tests: valid config → script tag injected with correct `src`, unknown app → throws, invalid config → throws
- [ ] `pnpm build` and `pnpm test` pass

## Technical Details

The manager reads the config from a known DOM element (e.g., `<script type="application/json" id="alistigo-config">`) or accepts it directly via `initArtifactManager(config)`. The playground uses the direct API.

Build: use `vite.config.umd.ts` pattern (same as `alistigo-artifact-list`).
