# Task: Verify all @m1 scenarios pass end-to-end

## Description

Run the full `@m1` Gherkin suite against the refactored `alistigo-artifact-playground`
and confirm every scenario is green. Fix any regressions before marking M2 done.

This is the acceptance gate for M2.

## Acceptance Criteria

- [ ] `pnpm test` (or `nx run alistigo-features-runner:test`) runs the `@m1` suite
- [ ] All `@m1` scenarios in `packages/alistigo-features/features/core/` pass:
  - `display-list.feature`
  - `add-element.feature`
  - `delete-element.feature`
  - `persist.feature`
- [ ] No CSP violations in the browser console during scenario runs
- [ ] Replay-equivalence property holds: replaying the event log produces the same projection at the end of every scenario
- [ ] `pnpm typecheck && pnpm nx qa:lint` clean across all M2-affected packages

## Technical Details

Run in CI order:
1. `nx run alistigo-artifact-playground:build` (or `dev`)
2. `nx run alistigo-features-runner:test --filter=@m1`

If any scenario fails, investigate whether the failure is in the runner URL config, CSP
blocking, or a genuine regression in the list artifact. Fix in the relevant task's package.

- Task 008: Gherkin runner must target the new iframe page URL
