# Task: Scaffold Astro app + Nx wiring

## Description

Create the `apps/personal-website/` Astro 5 project inside the Holos monorepo, starting from the `astro-bento-portfolio` theme as the base. Wire it into Nx with a `project.json` and ensure it integrates with the existing monorepo toolchain (pnpm workspace, Biome, TypeScript strict).

## Acceptance Criteria
- [ ] `apps/personal-website/` exists with Astro 5 installed
- [ ] `astro-bento-portfolio` theme files are present as starting point (not placeholder scaffolding)
- [ ] `project.json` has targets: `build`, `dev`, `typecheck`, `lint`
- [ ] `tsconfig.json` extends `../../tsconfig.base.json`
- [ ] `nx run personal-website:build` succeeds and outputs static files to `dist/`
- [ ] `nx run personal-website:dev` starts the dev server on a local port
- [ ] Biome is configured (inherits root `biome.json`)
- [ ] `astro.config.ts` sets `output: 'static'` and `site: 'https://mikael-labrut.alistigo.com'`
- [ ] Tailwind CSS v4 is installed and configured
- [ ] `public/CNAME` contains `mikael-labrut.alistigo.com`

## Technical Details

Clone / copy files from `astro-bento-portfolio` (GitHub: `Lol3rrr/astro-bento-portfolio` or similar bento portfolio theme from astro.build/themes). Remove placeholder personal data — keep the layout/component structure as the skeleton.

Nx `project.json` targets:
```json
{
  "build": { "command": "astro build", "outputs": ["{projectRoot}/dist"] },
  "dev":   { "command": "astro dev" },
  "typecheck": { "command": "tsc --noEmit" },
  "lint": { "command": "biome check src/" }
}
```

The `data/resume.json` lives at the repo root `data/` — the app should reference it via a relative import path at build time.
