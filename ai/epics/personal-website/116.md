# Task: Resume data file + TypeScript types

## Description

Populate `data/resume.json` with Mikael's actual career data (from LinkedIn export via the CLI from task 002, or manually). Define TypeScript types for the JSON Resume schema in the personal-website app. Ensure the data is complete enough for all sections (Experience, Skills, About, Hero).

## Acceptance Criteria
- [ ] `data/resume.json` exists at repo root with real data (not placeholder)
- [ ] All required JSON Resume fields populated: `basics`, `work` (≥3 entries), `education`, `skills` (≥10 entries)
- [ ] `apps/personal-website/src/types/resume.ts` exports TypeScript interfaces matching JSON Resume schema
- [ ] Import of `data/resume.json` in Astro is type-safe (no `any`)
- [ ] `basics.profiles` includes LinkedIn and GitHub entries

## Technical Details

TypeScript types file (`src/types/resume.ts`) should define:
```ts
export interface ResumeBasics { name: string; label: string; email: string; summary: string; ... }
export interface ResumeWork { name: string; position: string; startDate: string; endDate?: string; summary: string; highlights: string[] }
export interface Resume { basics: ResumeBasics; work: ResumeWork[]; education: ...; skills: ...; }
```

The Astro pages import `data/resume.json` with TypeScript's `resolveJsonModule` — the types are used for assertions.
