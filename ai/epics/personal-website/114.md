# Task: LinkedIn to JSON Resume CLI

## Description

Create a CLI app at `apps/linkedin-to-resume/` that takes a LinkedIn data export ZIP file and converts it to `data/resume.json` (JSON Resume format). Uses Clipanion + Bun, following Europa's TS CLI conventions (`ts-cli` skill pattern).

## Acceptance Criteria
- [ ] `apps/linkedin-to-resume/` project exists with `project.json` Nx targets
- [ ] `tsconfig.json` extends `../../tsconfig.base.json`
- [ ] CLI has a `convert` command: `bun run src/index.ts convert <path-to-linkedin-export.zip> [--output data/resume.json]`
- [ ] Command parses LinkedIn CSV files from the ZIP: `Positions.csv`, `Education.csv`, `Profile.csv`, `Skills.csv`
- [ ] Outputs a valid JSON Resume 1.0 schema document
- [ ] Writes output to `data/resume.json` by default (relative to repo root)
- [ ] CLI prints a summary of what was parsed (N jobs, N skills, N education entries)
- [ ] TypeScript strict mode, no `any`

## Technical Details

LinkedIn data export (from Settings → Data privacy → Get a copy of your data) is a ZIP containing CSV files. Key files:
- `Positions.csv`: work history (Company Name, Title, Description, Started On, Finished On)
- `Education.csv`: schools (School Name, Degree Name, Start Date, End Date)
- `Profile.csv`: name, headline, summary, location
- `Skills.csv`: skill names

JSON Resume schema target (jsonresume.org/schema):
```ts
interface Resume {
  basics: { name, label, email, summary, location: { city, countryCode }, profiles: [{network, url}] }
  work: [{ name, position, startDate, endDate, summary, highlights }]
  education: [{ institution, studyType, area, startDate, endDate }]
  skills: [{ name, level, keywords }]
}
```

Use `adm-zip` or `yauzl` for ZIP extraction, `csv-parse` for CSV parsing.
