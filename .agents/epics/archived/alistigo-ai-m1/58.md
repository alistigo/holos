# Task 002 — Document format: JSON-LD schema + serializer

## Goal

Implement `packages/alistigo-document-format` — the public, versioned `AlistigoDocument` JSON-LD format and the bidirectional `DocumentSerializer` that converts between it and the internal domain model.

## Reference

- [Domain model — Document Format section](../../../projects/alistigo-ai/domain/index.md)
- PRD F9 (JSON-LD, schema.org), N3 (human-readable), N4 (deterministic), N5 (interaction log)

## Deliverables

```
packages/alistigo-document-format/
  src/
    schema/
      alistigo-document.ts     # AlistigoDocument TypeScript type
      alistigo-event-record.ts # EventRecord shape in JSON-LD
      alistigo-element-item.ts # ItemListElement shape
    serializer/
      document-serializer.ts   # serialize(List) → AlistigoDocument
                               # deserialize(AlistigoDocument) → List
    validator/
      document-validator.ts    # validate(unknown) → AlistigoDocument | ValidationError
    constants.ts               # @context, vocab URIs, SCHEMA_VERSION
    index.ts
```

### JSON-LD shape (see domain model for full example)

- `@type: "ItemList"` at root
- `alistigo:listId`, `alistigo:schemaVersion`
- `itemListElement[]` — current Projection (position 1-based)
- `alistigo:listEventLog[]` — full event stream

### DocumentSerializer rules

- `serialize(list: List): AlistigoDocument` — takes List with uncommitted events appended to log, produces JSON-LD
- `deserialize(doc: AlistigoDocument): List` — replays `listEventLog` through `List.applyEvent()` to rehydrate aggregate
- Round-trip invariant: `deserialize(serialize(list))` must yield equivalent List state

### DocumentValidator

- Validates required fields: `@type`, `alistigo:listId`, `alistigo:schemaVersion`, `alistigo:listEventLog`
- Returns typed `AlistigoDocument` or `ValidationError`

## Acceptance

- `serialize → deserialize` round-trip test: same `listId`, same `elements`, same event count
- Validator rejects documents missing required fields
- Output JSON is valid JSON-LD (no structural errors)
- No imports from UI, application, or infrastructure layers
