# Task 007 — Acceptance runner: Playwright + Gherkin bridge

## Goal

Wire `packages/alistigo-features/` with a Playwright-backed Gherkin step runner so that all `@m1` scenarios can be executed against a real browser. M1 is done when every `@m1` scenario passes green.

## Reference

- PRD F7 (acceptance runner reads `.feature` files, validates `@m1` programmatically)
- PRD Success Criteria

## Deliverables

```
packages/alistigo-features/
  steps/
    core/
      display-list.steps.ts
      add-element.steps.ts
      delete-element.steps.ts
      persist.steps.ts
    world.ts              # Playwright browser + page fixture shared across steps
  runner/
    runner.ts             # entry point: parse .feature files, run @m1 tag filter
  playwright.config.ts
  package.json            # @cucumber/cucumber or similar Gherkin runner
```

### Step implementation approach

- Use `@cucumber/cucumber` (or `playwright-bdd`) for step definitions
- Each step file imports `world.ts` for browser/page access
- `world.ts` sets up a local HTTP server serving the widget bundle from `packages/alistigo-widget/public/`
- Before each scenario: fresh browser context + new `ListId` to avoid state leakage

### Key steps to implement

| Step | Action |
|------|--------|
| `Given an empty list` | Navigate to widget with fresh `list-id` |
| `Given a list: \| text \|` | Navigate and pre-populate via `document` attribute (JSON-LD) |
| `When I add "X"` | Fill add form + submit |
| `When I delete "X"` | Click delete on matching element row |
| `When I reload the list` | `page.reload()` |
| `Then I see "X"` | Assert element with text "X" visible |
| `Then I do not see "X"` | Assert element absent |

### Tag filtering

- Runner must support `--tags @m1` to run only M1 scenarios
- CI command: `pnpm --filter alistigo-features test:acceptance --tags @m1`

## Acceptance

- `pnpm --filter alistigo-features test:acceptance --tags @m1` exits 0 with all scenarios green
- Each scenario runs in isolation (no shared browser state)
- Output includes scenario name + pass/fail per step
