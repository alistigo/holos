# Task 008 — AI integration guide: llms.txt + MCP description

## Goal

Publish a machine-readable guide that teaches any AI how to produce a valid `AlistigoDocument` and embed the `<alistigo-list>` widget — without custom prompt engineering from the user (PRD F10).

## Reference

- PRD F10 (machine-readable skill, publicly hosted, web-discoverable)
- PRD F9 (JSON-LD document format)
- [Domain model — Document Format section](../../../projects/alistigo-ai/domain/index.md)

## Deliverables

```
packages/alistigo-widget/public/
  llms.txt                  # llms.txt-spec integration guide
  mcp-tool.json             # MCP tool description (JSON Schema)
```

### llms.txt

Following the [llms.txt spec](https://llmstxt.org). Include:

- What Alistigo is (one paragraph)
- When to use it (user asks to create a list / shareable list artifact)
- How to produce an `AlistigoDocument` — step-by-step with a minimal valid example
- How to embed the widget (`<script>` tag + `<alistigo-list document="...">`)
- Schema reference: required fields, optional fields, vocab URIs
- Constraints: M1 supports text elements only; one list per embed

### mcp-tool.json

MCP tool definition so an AI agent can call Alistigo programmatically:

```json
{
  "name": "create_alistigo_list",
  "description": "Create a shareable list artifact as an AlistigoDocument",
  "inputSchema": {
    "type": "object",
    "properties": {
      "title": { "type": "string" },
      "elements": {
        "type": "array",
        "items": { "type": "string" }
      },
      "actor": { "type": "string", "enum": ["user", "llm", "host", "system"] }
    },
    "required": ["elements"]
  }
}
```

### Hosting

- Both files served as static assets from `packages/alistigo-widget/public/`
- `llms.txt` accessible at the widget's root URL (conventional path per spec)

## Acceptance

- `llms.txt` is valid per the llms.txt spec (no broken sections)
- An AI given only `llms.txt` can produce a minimal valid `AlistigoDocument` (manually verified)
- `mcp-tool.json` passes JSON Schema meta-schema validation
