# Task 003 — Gherkin acceptance specs (@m1 scenarios)

## Goal

Write the complete `@m1` Gherkin acceptance scenarios in `packages/alistigo-features/`. These are the definition of done for M1. All scenarios must be tag-gated with `@m1` and follow the project's Gherkin step conventions.

## Reference

- PRD User Stories section (display-list, add-element, delete-element, persist)
- PRD Success Criteria
- [Domain model Ubiquitous Language](../../../projects/alistigo-ai/domain/index.md)

## Deliverables

```
packages/alistigo-features/
  features/
    core/
      display-list.feature
      add-element.feature
      delete-element.feature
      persist.feature
```

### display-list.feature

- Empty list renders with no elements visible
- List with pre-loaded Document renders its elements in insertion order
- Element text is visible and unchanged

### add-element.feature

- User adds a text element → it appears at the end of the list
- Adding a duplicate text value is allowed (different identity)
- Attempting to add an empty string → element is not added (validation error shown)

### delete-element.feature

- User deletes an element → it disappears from the list
- Other elements are unaffected (order preserved)
- Deleting an already-deleted element is a no-op / graceful error

### persist.feature

- After adding elements, reloading the page shows the same elements
- After deleting an element, reloading the page confirms deletion persists
- A fresh session with no stored Document shows an empty list

## Step conventions

- Use domain terms from the Ubiquitous Language glossary (List, Element, not "item" or "entry")
- Tables for collections: `Given a list:` + headerless table
- `When I add "X"` not `When I add an element "X"`
- Opening the widget is implicit unless the scenario is testing the open action itself
- `Then I see "X"` for element visibility checks

## Acceptance

- All scenarios are tagged `@m1`
- Feature files parse without errors
- Step definitions are stubbed (pending) so the runner can detect them — full implementation in Task 007
