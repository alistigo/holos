# Task: Scaffold @alistigo/artifact-posthog-plugin

## Description

Create `packages/alistigo-artifact-posthog-plugin` (npm `@alistigo/artifact-posthog-plugin`),
implementing `AlistigoPlugin` from `@alistigo/artifact-plugin-api`. This replaces the
PostHog logic currently in `packages/alistigo-artifact-list/src/analytics.ts`.

## Acceptance Criteria

- [ ] Package scaffolded, depends on `@alistigo/artifact-plugin-api` (type-only import) and `posthog-js`
- [ ] Default export is an `AlistigoPlugin` object: `name: "@alistigo/artifact-posthog-plugin"`
- [ ] `setup(ctx)`: reads API key from this plugin's own build-time env var (`VITE_POSTHOG_KEY`), no-ops silently if absent; otherwise calls `posthog.init(key, { api_host: "https://eu.i.posthog.com", persistence: "memory", autocapture: false, capture_pageview: false })`; idempotency guard so repeated `setup()` calls don't re-init
- [ ] `setup(ctx)` also calls `ctx.on("widget:displayed", (payload) => posthog.capture("widget_displayed", payload))`
- [ ] Own `vite.config.ts`: bundled dependency-free ESM output (`build.lib.formats: ["es"]`, nothing externalized — `posthog-js` must be bundled in)
- [ ] Unit tests: `setup()` is a no-op without an API key; `setup()` initializes PostHog exactly once across repeated calls; the `widget:displayed` handler calls `posthog.capture("widget_displayed", ...)` with the received payload (mock `posthog-js`)
- [ ] `pnpm build && pnpm build:typecheck` pass; built `dist/index.js` contains no bare `import` specifiers
- [ ] `pnpm nx qa:lint` passes

## Technical Details

Reference the existing `packages/alistigo-artifact-list/src/analytics.ts` for the exact
`posthog.init()` options and idempotency-guard pattern to preserve (EU host, memory
persistence, autocapture/pageview disabled — per ADR 0010's GDPR-no-consent-banner
rationale, which must not regress).
