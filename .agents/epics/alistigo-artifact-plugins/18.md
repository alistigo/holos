# Task: Genericize debug.ts's about() for loaded plugins

## Description

`packages/alistigo-artifact-list/src/debug.ts` currently exposes `version()`/`about()`
diagnostics that hardcode monitoring (Sentry) and analytics (PostHog) status fields by
reading `isMonitoringEnabled()`/`getMonitoringRelease()`/`isAnalyticsEnabled()`/
`getAnalyticsHost()` from the now-deleted `monitoring.ts`/`analytics.ts`. Since task 005
deletes those modules, `about()` must be genericized to report the set of currently
loaded plugins instead.

## Acceptance Criteria

- [ ] `debug.ts` no longer imports anything from the deleted `monitoring.ts`/`analytics.ts`
- [ ] `about()` reports a generic list of loaded plugin names (and, where cheaply available, each plugin's self-reported status) instead of hardcoded monitoring/analytics sections
- [ ] The plugin runtime (from task 001/005) exposes whatever minimal introspection `about()` needs (e.g. the list of successfully loaded plugin names) — add this to `createPluginRuntime`'s return shape if not already present, rather than reaching into private state
- [ ] Existing `version()` behavior (unrelated to plugins) is unchanged
- [ ] Unit test: `about()` output reflects whichever plugins were actually loaded in a given `mount()` call
- [ ] `pnpm build && pnpm build:typecheck` pass
- [ ] `pnpm nx qa:lint` passes

## Technical Details

Read the current `debug.ts` before editing — `about()`'s existing aggregation pattern
(one object combining several state sources) is the right shape to keep, just fed from
the plugin runtime instead of the two deleted modules.

## Dependencies

- Task 005 (deletes `monitoring.ts`/`analytics.ts`, introduces the plugin runtime this task reads from)
