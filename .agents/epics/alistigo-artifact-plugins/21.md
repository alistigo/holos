# Task: Gherkin features for Sentry + PostHog plugins

## Description

Add two new `.feature` files to `packages/alistigo-features/` covering the Sentry and
PostHog plugins' behavior at the same Application-layer/deterministic level the rest of
the suite uses — no real network calls. This requires a small taxonomy addition first:
a new group tag `@artifact-plugins` and a new milestone tag `@platform` (since this work
is orthogonal to the numbered `@m1`–`@m4` sequence already in `tags.ts`/`docs/tags.md`).

## Acceptance Criteria

- [ ] `packages/alistigo-features/src/tags.ts`: `GROUP_TAGS` gains `@artifact-plugins`; `MILESTONE_TAGS` gains `@platform` (documented as "cross-cutting platform capability, not tied to a numbered milestone")
- [ ] `packages/alistigo-features/docs/tags.md` updated to match (kept in parity with `tags.ts` per the existing validator)
- [ ] New folder `packages/alistigo-features/features/artifact-plugins/` with a `README.md` (per the one-folder-per-group convention, matching `features/core/README.md`'s style)
- [ ] `features/artifact-plugins/sentry-error-capture.feature`: tagged `@platform @artifact-plugins @capability:plugins`; scenarios cover (a) the plugin initializing when a DSN is configured, (b) an uncaught render error being reported through the `error:uncaught` subscriber, (c) silent no-op when no DSN is configured
- [ ] `features/artifact-plugins/posthog-analytics-tracking.feature`: tagged `@platform @artifact-plugins @capability:plugins`; scenarios cover (a) `widget_displayed` captured on first mount, (b) no double-report on remount, (c) silent no-op when no API key is configured
- [ ] Both files validate against the repo's Gherkin validator/linter (`gherklin.config.ts` / `tools/validate.ts`)
- [ ] Scenarios pass via the existing Application-layer runner, using fake/in-memory `AlistigoPlugin` test doubles — not real Sentry/PostHog network calls
- [ ] All pre-existing `@m1`/`@core` scenarios still pass (no regression)

## Technical Details

Read `packages/alistigo-features/features/core/display-list.feature` for the exact
Given/When/Then phrasing style and tag-ordering convention to match, and
`packages/alistigo-features/docs/style-guide.md` for the writing rules (declarative
steps, tables for collections, one-scenario-one-outcome).
