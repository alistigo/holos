# Task: ADR 0016 — composable artifact plugin system

## Description

Write `docs/adrs/0016-artifact-plugin-system.md` documenting the decision to structure
`@alistigo` artifacts with a composable plugin system: the unified interface + event bus
design, the choice to distribute plugins as independently-versioned npm packages, and
the choice to load them dynamically via `await import()` against a CDN-resolved URL
rather than bundling at build time. Update `docs/adrs/README.md`'s index table.

## Acceptance Criteria

- [ ] `docs/adrs/0016-artifact-plugin-system.md` created, following the exact template used by `docs/adrs/0008-error-monitoring.md`: YAML frontmatter (`status`, `date`, `deciders`), `## Context` (with a requirements table), `## Decision`, `## Rationale` (with a comparison table against alternatives), `## Consequences` (Positive / Negative-tradeoffs), `## Alternatives considered`
- [ ] Context section explains the duplication-avoidance motivation and explicitly notes ADR 0008/0010 are not being revisited (tool choices stand; only the wiring mechanism changes)
- [ ] Decision section covers: the `AlistigoPlugin` interface + `PluginEventBus` shape, npm-package distribution, `await import()` against a jsDelivr `@0`-pinned URL (reusing ADR 0011's versioning convention), and the host-owned generic `ErrorBoundary` + `error:uncaught` event design (replacing the `@sentry/react`-provided boundary)
- [ ] Rationale/comparison table considers at least: this design vs. a middleware-array-only approach vs. a Webpack-Tapable-style hook-only approach vs. bundling plugins at build time instead of dynamic-importing them
- [ ] Consequences section notes the accepted tradeoffs: `mount()` becomes async (breaking change to the "explicit mount" API), a narrower Sentry error-capture window (init moves from module-load time to inside `mount()`), and the new "bundled dependency-free ESM" build pattern plugin packages must follow
- [ ] `docs/adrs/README.md`'s index table gets a new row for `0016`
- [ ] Naming-conflict resolution with `.agents/prds/alistigo-ai-m3.md` is documented (or cross-referenced to task 010's PRD edit)
