# Task: GSAP ScrollTrigger + Lenis scroll animations

## Description

Add scroll-driven animations to all bento tiles using GSAP ScrollTrigger. Each tile fades in and slides up as it enters the viewport. Add Lenis for smooth scroll behavior. Respect `prefers-reduced-motion`.

## Acceptance Criteria
- [ ] Lenis smooth scroll is initialized on page load
- [ ] GSAP ScrollTrigger is set up to work with Lenis (ticker sync)
- [ ] Each `.bento-tile` animates: `opacity: 0 → 1` + `translateY: 30px → 0` when entering viewport
- [ ] Tiles in the same row stagger (100ms delay between tiles)
- [ ] `@media (prefers-reduced-motion: reduce)` disables all animations (tiles visible immediately)
- [ ] Animations are initialized only client-side (Astro `<script>` tag, not SSR)
- [ ] No layout shift during animation (tiles start invisible but reserve space via CSS `visibility`)
- [ ] Lenis scroll does not break native anchor link scrolling

## Technical Details

Lenis + GSAP ticker integration:
```ts
import Lenis from 'lenis';
import { gsap } from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';

gsap.registerPlugin(ScrollTrigger);

const lenis = new Lenis();
lenis.on('scroll', ScrollTrigger.update);
gsap.ticker.add((time) => lenis.raf(time * 1000));
gsap.ticker.lagSmoothing(0);
```

Tile animation:
```ts
gsap.utils.toArray('.bento-tile').forEach((tile, i) => {
  gsap.from(tile as Element, {
    opacity: 0, y: 30,
    duration: 0.6, ease: 'power2.out',
    scrollTrigger: { trigger: tile as Element, start: 'top 85%' },
    delay: (i % 3) * 0.1 // stagger per row
  });
});
```

Reduced motion: check `window.matchMedia('(prefers-reduced-motion: reduce)').matches` before init.

This script lives in `src/scripts/animations.ts`, loaded via Astro `<script>` in `Layout.astro`.
