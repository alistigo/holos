# Task: Projects + GitHub activity tiles

## Description

Implement two tiles that consume live GitHub data fetched at build time from the public API:
1. `BentoProjects.astro` — cards for top GitHub repos (by stars) for user `MLKiiwy`
2. `BentoGitHub.astro` — GitHub profile stats tile (followers, public repos count, avatar)

## Acceptance Criteria
- [ ] `src/data/github.ts` exports async functions that fetch GitHub API data at build time
- [ ] `BentoProjects.astro` renders cards for top 6 public repos: repo name, description, language, star count, link
- [ ] `BentoGitHub.astro` renders: GitHub avatar, username, bio, follower count, public repo count, link to profile
- [ ] API calls are typed (TypeScript, no `any`)
- [ ] Build does not fail if GitHub API rate-limits (graceful fallback to empty state)
- [ ] Data is baked into static HTML at build time (no client-side API calls)
- [ ] Repos with `fork: true` are filtered out by default

## Technical Details

```ts
// src/data/github.ts
const GITHUB_USER = 'MLKiiwy';

export async function getTopRepos() {
  const res = await fetch(
    `https://api.github.com/users/${GITHUB_USER}/repos?sort=stars&per_page=10&type=owner`
  );
  if (!res.ok) return [];
  const repos = await res.json() as GitHubRepo[];
  return repos.filter(r => !r.fork).slice(0, 6);
}

export async function getProfile() {
  const res = await fetch(`https://api.github.com/users/${GITHUB_USER}`);
  if (!res.ok) return null;
  return res.json() as Promise<GitHubUser>;
}
```

All fetches happen in Astro page/component `---` frontmatter (server/build time).
