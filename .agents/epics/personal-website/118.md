# Task: Bento grid layout + Tailwind v4 design system

## Description

Implement the CSS grid layout system for the bento grid and configure Tailwind CSS v4 with the site's design tokens (color palette, typography scale, spacing). This is the foundation that all section tiles build on top of. Mobile-first: single column on mobile, bento grid on tablet/desktop.

## Acceptance Criteria
- [ ] `BentoGrid.astro` layout component that places tiles into the CSS grid
- [ ] `Layout.astro` wraps the page with `<head>`, fonts, global styles
- [ ] Tailwind v4 configured with design tokens: dark background (`#0a0a0a`), accent color, type scale
- [ ] Grid is responsive: 1 column < 768px, multi-column grid ≥ 768px
- [ ] Tiles support span variants (1-col, 2-col, full-width) via CSS grid `grid-column` spans
- [ ] Font loaded (e.g. Inter or Geist from Google Fonts / Fontsource)
- [ ] Global CSS resets and base styles in place
- [ ] Empty placeholder tiles render correctly at all breakpoints

## Technical Details

Grid system using CSS grid with named areas or `grid-template-columns`. Each tile is a `<div>` with a CSS class like `tile--span-1`, `tile--span-2`, `tile--full`.

Tailwind v4 config (`tailwind.config.ts` or CSS `@theme` block):
```css
@theme {
  --color-bg: #0a0a0a;
  --color-surface: #141414;
  --color-border: #262626;
  --color-accent: #6366f1;
  --color-text: #fafafa;
  --color-muted: #a1a1aa;
}
```
