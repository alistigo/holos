# Task: Implement Claude iframe simulation + CSP headers

## Description

Finalize the playground iframe to match Claude's production embedding environment as
closely as possible: correct sandbox attributes and a CSP policy on the iframe page that
mirrors Claude's, with a localhost exception for local development.

## Acceptance Criteria

- [ ] Iframe element in the host page has exactly these attributes:
  ```html
  <iframe
    style="zoom: 1; height: 100%; width: 100%;"
    sandbox="allow-scripts allow-same-origin"
    referrerpolicy="no-referrer"
    data-no-service-worker="true"
    allow="fullscreen; clipboard-write"
    src="..."
  />
  ```
  - No `class` attribute for sizing — only `style`
- [ ] CSP headers applied to the iframe page (`iframe.html`) via Vite dev server middleware:
  - Base policy mirrors Claude's `claudeusercontent.com` CSP
  - Adds `localhost` and `127.0.0.1` to `connect-src`, `script-src`, and `frame-src` where applicable
  - Policy is documented in a comment in `vite.config.ts`
- [ ] Artifact boots and renders correctly inside the sandboxed iframe (no console errors related to CSP violations)
- [ ] Verified in Chrome and Firefox

## Technical Details

Vite dev server middleware for CSP:
```ts
// vite.config.ts
server: {
  middlewares: [
    (req, res, next) => {
      if (req.url?.includes('iframe')) {
        res.setHeader('Content-Security-Policy', CSP_POLICY)
      }
      next()
    }
  ]
}
```

Claude's iframe sandbox uses `allow-scripts allow-same-origin` — this means no
`allow-top-navigation`, no `allow-forms`, no `allow-popups`. The artifact must not
require any of those.
