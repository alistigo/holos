# Task: Build host page — Dev Config Form + iframe layout

## Description

Build the host page of `alistigo-artifact-playground`: a split-vertical layout with the
Dev Config Form on the left and the artifact iframe on the right.

The Dev Config Form is a small React app that controls the iframe's config document.

## Acceptance Criteria

- [ ] Host page layout: full-viewport, split 50/50 vertically (left: form, right: iframe)
- [ ] **Generic controls** (all artifacts):
  - [ ] Artifact type selector — populated from `@alistigo/artifact-manager`'s artifact map; on change, reloads the iframe with the new artifact
  - [ ] Language selector — ISO locale code (en, fr, nl, ...); passed in config
  - [ ] AI chat context selector — fixed list: ["claude"]; reads-only for M2, included for future extension
  - [ ] Reload button — force-reloads the iframe (same config)
  - [ ] Clear data button — calls `localStorage.clear()` in the iframe context, then reloads
- [ ] **Dynamic per-artifact section** (below generic controls):
  - [ ] For `@alistigo/artifact-list`: `readonly` toggle (boolean, defaults to false)
  - [ ] Section is hidden / empty for unknown artifact types
- [ ] Config changes in the form update the iframe `src` or inject updated config without full page reload where possible
- [ ] The iframe element is wired up (attributes will be finalized in task 007)
- [ ] No CSS framework — plain CSS or CSS Modules; no Tailwind/shadcn for this internal dev tool

## Technical Details

The form communicates with the iframe via:
1. Encoding the config as a URL param on the iframe `src` (simple, works with `sandbox`)
2. Or injecting a `<script type="application/json">` into the iframe DOM before load

Pick the URL-param approach for M2 simplicity. The artifact-manager on the iframe page
reads config from `URLSearchParams`.

Form state is local React state — no global store needed.
