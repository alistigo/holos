# Task: Scaffold @alistigo/artifact-config-format

## Description

Create `packages/alistigo-artifact-config-format/` — the aggregate package that defines
the base config schema and combines all known per-artifact schemas into a single
discriminated union.

## Acceptance Criteria

- [ ] Package scaffolded via `bash scripts/new-package.sh alistigo-artifact-config-format`
- [ ] `package.json` name: `@alistigo/artifact-config-format`; depends on `@alistigo/artifact-config-list-format`
- [ ] Base config schema defined:
  ```json
  {
    "type": "object",
    "required": ["app"],
    "properties": {
      "app": { "type": "string" },
      "lang": { "type": "string" }
    }
  }
  ```
- [ ] Discriminated union schema: `if { "properties": { "app": { "const": "@alistigo/artifact-list" } } } then { <list schema> }`
- [ ] TypeScript type exported: `ArtifactConfig` (base) and `ListArtifactConfigFull` (base + list fields)
- [ ] Validation helper exported: `validateArtifactConfig(value: unknown): ArtifactConfig` — returns typed config, throws descriptive error when `app` is unknown or schema invalid
- [ ] Unit tests: valid list config, unknown `app` value throws with actionable message, missing required `app` throws
- [ ] `pnpm build` and `pnpm test` pass

## Technical Details

Adding a second artifact in a future milestone means:
1. Create `@alistigo/artifact-config-<name>-format` (leaf)
2. Add it as a dependency of this package
3. Add an `if/then` branch to the discriminated union here

Dependency direction is always: leaf → aggregate. Never import `artifact-config-format`
from a leaf artifact-config package.
